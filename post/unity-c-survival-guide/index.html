Unity C# Survival Guide

发布于 2020-05-04 | 分类于 Unity | 24分钟 | 6386字数

引子

Unity3D官方代码教程Survival Guide相关笔记。
C#语言是探索Unity技术世界的重要入门基础，UnityAPI是玩转引擎的重要手段。
本笔记根据官方教程顺序记录，不会很细，只针对一些难点进行描述。
不是太注重语法细节，重要的还是理解用法和原理，语法细节查菜鸟教程或者API都是很快的。
前两章基本上是在Unity的API当中可以查到相关的，后面的C#知识在微软的技术文档中能查。
这个只是常用基础+一些高级技巧初识，有些其他的也可以另行查阅学习。

Unity代码初级认识

Guide的第1、2章，主要是引入一些Unity最最基础的API。
因为视频是通过Unity引擎界面来直接引导教学C#的，所以需要知道一些基本的操作。
主要是键鼠输入，操作物体，旋转物体的API。

Quick Tips and Assets 快速开始

改变物体位置：transform.x/y/z
键鼠输入：Input.getKeyDown，Input.getAxis
移动物体：transform.Translate(位移向量Vector3 * 速度 * Time.deltaTime)
实例化/销毁物体：Instantiate/Destroy
暂停游戏：Time.timescale设置为0-1之间
Console Debug：Debug.Log();
PS：团队合作和Assets部分比较特殊，未作记录，略

Rotations 物体旋转

Euler Angle：欧拉角，物体关于x，y，z轴旋转的角度
Quaternions：四元数，物体旋转的描述数(x,y,z,w)
Gimbal Lock：万向节死锁，按照动态欧拉角（环架）体系旋转时，物体旋转后出现的坐标轴重叠导致损失一个方向上的自由度。避免出现该锁问题的方法就是利用四元数来表示旋转。
Quaternions.identity：常量，表示没有旋转，即四元数为（0，0，0，0）
Quaternions.LookRotation：物体正方向forward面向指定Vector3向量
Debug.DrawRay：Debug手段，画出一条射线，可以指定Color类颜色常量

Slerp(Spherical linear interpolation)：球形线性插值，意思是符合一定的数学线性插值法，使得旋转有渐进性，光滑曲面地缓慢移动，API为：

Quaternions.Slerp(四元数起点，四元数终点，时间浮点数（通常是Time.deltaTime）)

C#语法基础

Guide的第3-8章，C#入门语法基础，面向过程。大体上讲，和C、C++、Java都有极大的共通性，所以仅简述。有过其它语言学习经历的，可以很快上手。具体语法细节不列了。

Variables 变量

变量就是程序存储数据所开辟的空间，通过名字来访问和使用。
关键字：type变量类型如int。C#与C家族和Java的基本相似，具体可以查阅API。

值类型和引用类型

类型分为值类型和引用类型。
C#语言独立预设的类型都是值类型比如int，char这些最基础的。
根据类来创建的对象类型，即类型名为类名的变量都属于引用类型。比如C#预设的Double类就是double的封装类，就是引用类型，注意区别大小写。注意string也是引用类型。后文涉及内存管理会详述。
C#当中还有一些动态类型object，dynamic，var之类的，var比较常用。这些又被称为弱类型。
在Unity当中，类型type也可以是unity组件类的对象类型，比如GameObject，Transform。
另：Mathf.Round，数学库函数，截取整数部分，可以通过乘除10的多少次方控制处理有多少位小数.

If Statements 条件判断语法if

条件判断，作用是依照判断条件控制相应语句执行。
关键字：if else

Switch Statements 条件判断语法switch

同样是条件判断，适用于控制变量统一的多条件情况，更加直观的语法。
关键字：switch case default break

Loops 循环语法

循环，作用是依照循环控制条件多次执行语句。
关键字：while do-while for foreach

Arrays 数组

数组是变量的集合，同种类型的多个变量连续存储就成了数组
关键字：[]
数组声明的时候，在类型后面加上[]就可以了，如同C/C++当中的*号，道理相同
一维数组访问的时候语法格式：数组名[index]，其中index是下标
任何类型的数组也算是个类型，也可以成为参数、返回值等等。
在Unity中，由于public类型的都是面板可视化的（后文会讲），所以public的数组没有必要指定长度和具体内容元素，可以在Unity引擎当中直接设置。
Array类：所有数组的基类，提供了一些很有用的数组属性和方法。使用方法通过数组名和符号.访问。比如Length可以直接获取数组的长度。Sort可以对一维数组进行排序等等。其他的可以查阅API或菜鸟教程。
可以通过for、foreach来遍历数组进行相关操作
其他扩展知识点：

  • 多维数组 二维及以上的数组，每一维度长度固定
  • 交错数组 更加灵活的多维数组，每一维的长度都可以指定

Functions and Methods 函数和方法

在程序语言中，本质上函数和方法是同一个东西，是具有一定功能并被组织起来的的代码段。
函数是方法的数学描述，方法是函数的现实意义。
由于是一个数学概念，所以有三大要素：参数，过程代码，返回值
依照以下格式编写。使用的时候直接函数名(参数)的方式调用并返回相应类型的返回值。
语法格式：[权限] [返回值类型] [函数名](参数){代码段}
关键字：ref out return

Classes 类与对象（C#面向对象）

Guide的第9章，C#进阶语法，面向对象
C#采用的是OOP模式，也就是说它是一门面向对象的程序语言
而在Unity引擎中，结合了传统的类和Unity引擎机制特别处理的类。
接下来，将解析类与对象各概念模块。主要语法样例：

public abstract class Unit : MonoBehaviour
{
    //场景脚本类衔接
    public BattleController BattleController;
    public GameObject GameObject;//场上图标
    public string Name;//单位名字
    public int HP;//生命值
    public int ATK;//攻击力
    public int DEF;//防御力
    public int DEVELOP;//发育值
    void Start(){}
    void Update()//作战单位数据刷新,死亡判定,被动技能监听
    {
        GameObject.GetComponentInChildren<Text>().text = string.Format("{0}",HP);
        if (HP <=0) Destroy(GameObject);//死亡
    }
    public abstract void ActiveSkill(Unit Target);//主动技能
    public abstract void PassiveSkill();//被动技能
    public void Choose(){}//选中单位    
}
[Syetem.Serializable]
public class EarthGuard : Unit
{
    public EarthGuard(){}
    public override void ActiveSkill(Unit Target){}
    public override void PassiveSkill(){}
    ~EarthGuard(){}
}

大体说明：Unit类是抽象类代表游戏中的战斗单位，EarthGuard继承于Unit，代表具体某一种单位。

MonoBehaviour Class 行为类

所有和Unity场景当中物体相关的，涉及到继承使用Unity引擎环境提供的MonoBehaviour类相关API的类，就称为行为类。一句话概括就是，只要这个代码涉及到了任何Unity游戏引擎里面的东西，就要继承MonoBehaviour类才能开展相关操作。一般需要在游戏场景中实例化，进行各种活动的物体，就需要挂载这类脚本。比如你让游戏内物体移动，受一个力等。Unity引擎环境和微软C#语言的联系就在这么个MonoBehaviour上。
语法上，首先是引入命名空间UnityEngine以及继承MonoBehaviour类。在游戏引擎内创建脚本都会这么自动生成，如果要写定制类就手动删了吧。其次是还会自动生成两个函数start和update，简单来说行为类的初始化不写构造函数的，因为是挂载在游戏物体上的。可以在start当中进行初始化，这样每当挂载的物体在场景内出现，就会调用一次这个函数。update则是start接力版，每一帧调用一次，可以做周期性逻辑控制和并行处理的东西，比如控制当达成什么条件就实时。。。发个任务。。。之类的。其他用法见API，比如update还有一个稳定版本叫做fixedupdate。读者如果接触过多线程生命周期这种的，这很类似，相当于每个行为类在场景中有它的生命周期。

Custom Class 定制类

可以理解为设计者自定义的类所以叫做定制类，其实就是C#普通的类而已，为了和行为类区别开，也可以说是非行为类。设计者可以不用继承MonoBehaviour类（因为没有那个必要），专注于处理和场景以外的模块比如数据库操作，各种后台逻辑之类的。语法和C#类与对象完全一致。

类成员可视化

可视化属于一种Unity特质的C#特性，可使得变量或者类成员在检视面板可视化+直接手动操作。
关键字：[SerializeField] 和 [Syetem.Serializable]
前者针对行为继承类，后者针对定制类。
因为行为继承类默认可视化，但它的成员不是；所有public的变量都默认可视化。

访问权限

基础概念补充，权限在之前也都有出现，代表着相关代码的作用域和被作用域。
一般来说，权限用公共public或者私有private，但是还有一种是保护级protected，这样那些相关继承类都可以访问得到，基本理解就是介于上述两者之间的一种部分权限开放。C#没有友元这种权限。

虚函数和重写

这是继承和派生的实现方式。虚函数出现在基类中，重写出现在子类中。
虚函数virtual，意味着子类可以重写；重写override，意味着重写了基类的虚函数

结构体

关于结构体，在C#当中和C++一样，结构体和类都是可以使用的（Java没有结构体）。
结构体语法和类基本一致，关键字为struct，对应class。
注意，结构体属于值类型存储在Stack栈上，注重存储性能；类的对象属于引用类型存储在Heap堆上，注重功能完善。引用类型作为类与对象的实践，它是指针思想的继承者，因为多数情况下比较大，所以通过引用来节约拷贝空间。
值类型和引用类型的内存分配语法区别是，是否需要new来在堆上申请空间。

Guide的第10-18章，C#高级编程知识点
从这里开始是这个教程我觉得正文开始的地方。这里对于之前的两个大模块有很多补充
涉及到了许多不那么易懂的部分，并且开始涉及数据库、数据结构等计算机专业知识

Static Types 静态类型

语法样例：
静态成员变量：public static int a;
静态方法：public static void Main(string args[]){}
静态构造函数： static A{}
关键字：static
关于静态类型，教程本身没有很长的介绍，以下简述使用思路和语法技术要点
类似于C++，C#的静态类型就是存在于类当中的固件，只要整个程序在运行，它们都将存在，因为本质上静态类型的变量和方法是在堆上面申请的空间，因此将一直存在直到整个程序结束。所以它们具有着“独一份”的特质，因此不会也不能被实例化。类当中的静态成员将被每个对象共享，其实可以理解为静态成员在另一个次元，没有生老病死2333。
我们可以利用这一特性进行很多方便的设计。首先明确，类，变量，函数都可以是静态的。静态的类是那种工具类；静态变量可以用来记录那种全局数据，比如说成就之类的，其实很多常规数据用这个也更加清晰切容易访问；静态函数我将其称为“上帝操作”，典型的就是进行系统消息提示，可以将这个功能写成一个静态函数，参数是消息。这样在任何地方需要消息输出，直接用就可以了。
静态变量的初始化，可以直接赋值，但是建议在对应的类当中实现静态构造函数，例如;其实呢就是说把构造函数的public换成了static。然后在这个翻版的构造函数当中处理所有静态东西。
注意：静态类当中不能有非静态成员，因为它不会被实例化；静态函数当中也不能访问调用任何非静态成员；静态变量需要被初始化好。

Properties 属性

语法样例：

public class Elf{ // 精灵类
    private int hp; //真正的HP变量，private存数据
    public int HP{ // 属性，变量的外壳，控制访问
        get{ //读
            return hp; 
        }
        set{ //写，hp上限设定为30，下限设定为0
            if(hp + value > 30) hp = 30;
            else if(hp + value < 0) hp = 0;
            else hp += value;
        }
    }
}

简单来说，属性是变量的一种扩展形式，关注到了“访问权限”。
在C++和C#当中，都有public或者private这种来进行权限管理，这是一个办法，但这是很简单的控制可不可以访问。而我们经常遇到一些特殊访问控制，比如增加HP但不可以超过上限，超过就溢出的处理。这时候就是属性工作的时候了。
以前做数据结构ADT题的时候，C++建议把成员变量声明为private，然后通过函数才可以访问。C#的属性其实就是组合了一下这些，成为了一个语法糖。

Namespaces 命名空间

语法样例：常用命名空间

using System.Collections;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using UnityEngine;
using UnityEngine.UI;

关键字：创建namespace [空间名] {代码}；使用using [空间名];
在C++当中也存在了，Java当中则是那些库或者package包，在C#当中基本完全充当了头文件的作用，封装了预设的代码，同时为了避免重名相当于加了看不见的前缀名。
在开发的时候可以用来开发工具类代码，增强多人协作的操作性和软件复用度。往往研发岗实现功能后可以将研发的代码写在一个命名空间内，然后交付开发人员使用它来开发。
C#自带的命名空间有像最基础自动生成的两个System.Collections和Genetics，基本覆盖基础语法功能和一些高级设计技巧组件比如泛型和集合框架，一些基础数据结构和算法等。
Unity自带的命名空间有像最基础默认的UnityEngine用来支持引擎提供的MonoBehaviour,即代码文件开头自动生成using UnityEngine;，再多举个常用例子比如UnityEngine.UI是支持引擎内UI的。

Lists 线性表

语法样例：

public List<int> data = new List<int>();

关键字：List
上面这句话声明了一个线性表，即泛型类List< T >对象data，其中T是int代表这个线性表装的都是int

泛型与集合框架

泛型的知识此处不详讲，简单来说就是当类中相关内容的类型并不是该类要关心的点时，可以通过设置泛型动态选择以及识别类中的涉及的一些类型。具体的请自行参考学习，这个点很重要。
集合框架是C#提供的一些简单数据结构的实现，可以理解为封装好的数据结构类，类似于C++的STL。上面命名空间已经讲了支持它的命名空间，一个支持普通型，一个支持泛型。
以下就要讲的List就是支持泛型的一种集合框架，对应线性表（链表）这种数据结构。

ArrayList和List< T >

C#官方教程讲的是后者，因为后者是类型安全而且好用的，一般也用后者。
ArrayList是普通的线性表，可以理解为一个动态的数组，不需要指定长度，中间删除了元素后整个表会自动收缩，不支持泛型但是同时允许表中数据随便是什么类型。因此它不是类型安全，因为不知道取出的数据是什么类型。比如说不能把随便一个对象做加法运算，因为不确保是否是数字。正因为如此，每次取出数据操作需要对变量拆箱，这将损耗程序性能。
ArrayList提供了很多对表的操作，比如在表尾加入一个数据，或者是返回表长度，清空整个表等。这些集合框架的方便之处就在此。
接下来讲List：List是主要常用的数据结构，线性表。它基本实现了大部分ArrayList能实现的功能方法，同时类型安全因为支持泛型，这意味着表中每一个元素都是同类型的，在创建表对象的时候需要设置类型。本质上来说，ArrayList和List都实现了一个决定线性表结构的接口，所以两者极其类似。
List可以通过索引下标[]访问，也支持迭代器IEnumerator访问，可以动态组织和排序数据，主要用for和foreach遍历。实用性很高，和C++的vector基本类似。

Enums 枚举

语法样例：

public enum Status{usingCard,target,ememy,waiting};//枚举行动玩家处于的阶段

关键字：enum
C家族语言枚举就是限定了范围的一种特殊变量，比如说性别只能是男或者女或者未知三种。本质是一个小的常量集合。使用的时候，可以通过每个常量的名字访问。要获取它们的整型数值，需要强制类型转换成int。
枚举也是值类型

Dictionary 映射集合（字典）

语法样例：

public Dictionary<int,string> data = new Dictionary<int ,string>();

关键字：Dictionary，KeyValuePair< TKey , TValue >
上面这句话声明了一个映射集合，也可以叫做散列表，即泛型类Dictionary< TKey , TValue >对象data，其中TKey是int代表这个映射的键是int类型的，TValue是string代表每个键对应的值是string类型的，即实际存储的值是string字符串

HashTable和Dictionary< TKey , TValue >

和List同理，C#官方教程讲的是后者，因为后者是类型安全而且好用的，一般也用后者。
HashTable是普通的散列表。散列表有别于线性表，主要用于快速查询，可以理解为是是通过键值对的方式存储的线性表，查询的时候与比下标索引更为好用的键Key。HashTable是线程安全的。
同样的，HashTable也提供了很多对散列表的操作，比如判断是否存在键Key。
Dictionary的道理正如同List之于ArrayList，就是HashTable的泛型版本，类型安全，但是并不线程安全，和HashTable的查询效率也不是绝对的好，即使不需要装箱拆箱。

访问方式，KeyValuePair< TKey , TValue >

关于访问和遍历的方式，Dictionary的花头稍微多一点。
首先，同样支持下标索引[]，支持for和foreach，动态组织数据。支持通过键key的内容查找返回值Value的内容，这样索引效率比线性表遍历查找要高。
因为是以键值对的形式存储，所以再for和foreach的时候，可以分为遍历键和遍历值。同时还提供了一个辅助的集合叫做键值对KeyValuePair，它封装了一对键值，可以用foreach直接遍历，通过访问成员key和value来得到具体数据。

Abstract Classes 抽象类

语法样例：

public abstract class Card : MonoBehaviour
{
    //场景脚本类衔接
    public BattleController BattleController;
    public GameObject GameObject;//场上图标
    public int price;//购买价格
    public void Use()//玩家使用卡牌的操作接口
    {
        StartCoroutine(Effect());return;
    }
    public abstract IEnumerator Effect();//玩家使用卡牌的效果
}
public class Develop : Card
{
    public override IEnumerator Effect()//发育牌效果:发育值+2
    {
        BattleController.ActivePlayer.DevValue += 2;
        BattleController.Text(string.Format("玩家{0}发育值+2", BattleController.ActivePlayer.Name));
        Destroy(GameObject);
        yield return 0;
    }
}

关键字：abstract，interface
这个样例和先前的有大致相像之处，Card作为抽象类代表卡牌，Develop继承抽象类并实现抽象方法，代表具体的一种卡牌，此处是发育牌。

抽象类和抽象方法

抽象是C和C++都没有的特质，Java有，C++虽然没有但是其实也能算有实现机制。
顾名思义，抽象就是“不具体实现”，有抽象类和抽象方法。
先说抽象类。一般来说，当设计的多个类有一部分共同特征，那么应当将这些共同特征抽象成为一个类，然后其他具备这些特质的类可以从它派生。这样可以加强代码设计、复用，有利于维护，避免代码多次重复。
接下来说说抽象方法。在前面讲类与对象的时候，没有重点写继承与派生的方法（函数）概念，只是简单地说了一下，对于普通类是由基类设置虚函数和子类重写虚函数来实现的。那么因为我们这是个抽象类，所以里面的元素可以是抽象概念，即方法可以不具体实现，等待继承的子类来实现。这就是抽象方法。
抽象类和抽象方法的声明就是加一个关键字abstract，子类在实现抽象方法的时候，也需要重写。
所以可以得到一个小结：对于要被重写的方法，普通类中就是虚函数，可以被先行实现一下；抽象类中就是抽象方法，不需要也不能被实现。
抽象设计更多地要注意这种类与对象设计理念。

接口

接口是面向对象语言独特的特质，C#和Java有这个。
可以说接口是“面向方法”的，它更纯粹，直接去对继承它的子类的方法进行一个约束，即继承该接口的类必须重写里面所有的方法。接口形式简洁，没有变量只有方法，可见抽象程度更高。
接口的一大关键好处是解决了C#只能单一继承的问题，即一个类是可以继承多个接口的。
接口需要直接定义权限，声明就是将类的关键字class用关键字interface代替。因此每个接口方法不必写权限以及abstract，但是继承它的类实现还是需要写override。
接口在编码上其实倒是没那么地便利，但它对设计和约束有很多好处。

Linq LINQ模块（数据库与集合框架）

Linq是微软在C#当中提供的一种数据库与集合框架的便利操作，就是避免去写SQL而是通过函数设置参数来查找。Java当中也是有如此设定的，例如安卓移动开发和JPA中都有。
使用之前需要引入命名空间System.Linq
预设现在有了一个集合类型var test;，这里的var是动态类型，实际使用时不确定的话可以是var，也可以是数组，列表等集合类型。Linq相关函数的调用方法并不是直接写名字，而是写集合名字.函数名，返回的就是操作结果。比如说any函数，那就是通过test.any()的方式调用，返回的就是对应结果。
Any：参数为Lambda表达式指定条件，返回bool类型，表示是否有任何一个满足条件的了结果。
Contains：参数为string字符串，返回bool类型，表示集合是否包含参数。
Distinct：没有参数，返回还是一个集合类型，将原来集合当中重复元素去除后得到的集合，即“去重”。SQL语言当中查询结果也有distinct关键字，道理差不多。
Where：参数为Lambda表达式指定条件，返回还是一个集合类型，也就是对应SQL的where条件查询语句，将所有符合条件的元素查找出来。
OrderBy：按升序对序列的元素进行排序。
OrderByDescending：按降序对序列的元素排序。
ThenBy：按升序对序列中的元素执行后续排序。
ThenByDescending：按降序对序列中的元素执行后续排序。
Reverse：反转序列中元素的顺序。
Average：求平均数。
其他更多函数和教程可以通过IDE语法功能或者查API。
