引言和分组加密工作模式

简单概念：分组密码就是将明文分成等长的每一组，然后分别处理加解密

分组密码设计原则

  • 针对安全性的一般设计原则：
    明文分组长度和密钥长度望尽可能大
    混乱原则：又称混淆原则，是指密钥和明文以及密文之间的依赖关系尽可能的复杂
    扩散原则：密钥或明文的每一位影响密文的许多位以便隐蔽明文的统计特性

两种经典算法结构

  • Feistel结构
    Feistel结构是典型的迭代密码.Feistel结构的解密与加密是完全一样的，除了所使用的子密钥的顺序正好相反。
  • SPN结构
    SPN结构也是一种特殊的迭代密码 。SPN结构和Feistel结构相比，可以得到更快速的扩散，但是SPN密码的加解密通常不相似。

分组加密工作模式

不同的工作模式，可以在形式上对加密效果产生影响，具有不同的适用场合

电子密码本模式（ECB）

即直接流加密，C[i] = Ek(M[i])
相同密钥作用下，在相同的明文加密后将产生相同的密文 ，易于暴露明文的固有格式
各密文块间缺乏相关性，信息易于受到块替换攻击

DES

SDES这里跳过，直接上完整基础的DES加解密算法。
DES是一种明文分组为64比特，有效密钥56比特，输出密文64比特的，具有16轮迭代的分组对称密码算法，DES由初始置换，16轮迭代，初始逆置换组成。
细节主要就是按下标的置换，异或，S盒置换，前两者比较简单。S盒置换不懂的可以搜一下。
Github上Github上有输入64位明文的算法实现，但是无分组工作模式。
以下描述DES算法具体内容

算法描述，结合上图阅读

初始输入：

64位原文，64位密钥

子密钥准备

核心算法的每一轮迭代，都需要一个子密钥Ki
生成方式：
将64位密钥进行EP1置换，缩短为56位结果

开始16轮密钥生成迭代

将56位结果，左28位作为高位，右28位作为低位
根据LOOP数组对应的值，高低位循环左移数位
然后将目前的56位结果进行EP2置换，生成为48位结果
这个结果就是子密钥Ki

经过上述操作的16轮迭代后，得到16个子密钥

核心算法

将64位原文作IP置换，得到64位结果

开始16轮迭代

将64位结果，左32位作为Li组，右32位作为Ri组，i+1为实际迭代轮数，所以i为0-15代表1-16轮

  • f函数 输入为Ri组
    将R组进行IP置换，扩展为48位
    将R组和子密钥Ki作异或XOR运算
    将48位异或结果，进入8个S盒，处理为32位
    将32位S盒结果作P置换，引起雪崩效应，起混淆作用
    结果记为f(Ri,Ki)
  • 异或交换
    Li+1 = Ri
    Ri+1 = Li异或f(Ri,Ki)，实际实现中处理了L和R的覆盖问题
    可见上述就是将没动的左半边和f函数结果进行了异或，然后左右交换

经过上述操作的16轮迭代后，得到最后64位结果
将64位结果的左L16右R16再次交换，其实就是抵消掉最后一轮的左右交换，实际实现中第16轮不交换
将交换恢复结果作IP逆置换，得到64位结果

最终输出：

64位结果

加解密的区别

DES的加解密很有意思，密钥生成和处理算法上是相同的
不同点仅在于，16个子密钥的使用顺序是完全相反的，加密从密钥1-16使用，解密时从密钥16-1使用
即每一轮使用的密钥不一样

改进方案：三重DES

套娃技术，素质三连算法迭代
加密顺序：加密——解密——加密
解密顺序：解密——加密——解密

AES

高级加密标准：AES 是DES的替代者。
1997年9月12日，NIST发布了征集算法的正式公告，要求AES具有128比特的分组长度，并支持128、192和256比特的密钥长度，而且要求AES要能在全世界范围内免费使用。
2000年10月2日，Rijndael算法被选择为高级加密标准。
Github上AES提供一个基础的AES加解密，还有基于CBC工作模式对于文件的加解密。
采取标准的，密钥长度128，10轮，扩展密钥长度176的最小组。
下面将先描述其数学基础，再进行大致的算法描述

数学基础

AES中的运算是按字节的，并把一个字节看成是系数在有限域GF(2)上的次数小于8的多项式（即把一个字节看成是有限域GF(2^8)中的一个元素) 。
多项式形式：b7x7 + b6x6 + b5x5 + b4x4 + b3x3 + b2x2 + b1x + b0，这些bi都是系数0或1
打个比方说57：(01010111)可写成： x6 + x4 + x2 + x + 1
因此，这涉及到抽象代数，在有限域上的运算

GF(2)加法

在多项式表示中，两个元素的和是一个多项式，其系数是两个元素的对应系数的模2加(即异或)。
例如：“57”和“83”的和为： 57⊕83＝D4
对应多项式记法：
57→01010111 → x6+x4+x2+x+1
83 →10000011 → x7+x+1
(x6+x4+x2+x+1)+(x7+x+1)＝x7+x6+x4+x2 →11010100 →D4 此处模2加法，遇到相同的就抵消

GF(2)乘法

不作证明，仅作描述
首先，有限域GF(2)上关键不可约分多项式：m(x)= x8+x4+x3+x+1
为啥关键呢，因为乘法运算最后要模它。十六进制是11B
例如：‘57’·‘83’=‘C1’可表示为以下的多项式乘法：
(x6+x4+x2+x+1)·(x7+x+1)=x7+x6+1(mod m(x))

性质

GF(2)的这种乘法满足交换律、分配律，且有单位元‘01’
满足类似贝祖等式：b(x)a(x)+m(x)c(x)=1即a(x)·b(x)=1 mod m(x)，a(x)是b(x)的乘法逆元

x乘法

这是对上面的补充，用适合计算机的方式来表达和运算。记作x*B(x) =
(b6 b5 b4 b3 b2 b1 b0 0)；b7 ＝0 当结果最高位是0，保留剩下7位贴0
(b6 b5 b4 b3 b2 b1 b0 0) ⊕(00011011)；b7 ＝1 当结果最高位是1，还要再异或一个数
所以，简单来说，a(x)·b(x)就可以像小学生那样，将a拆成一个个x的幂，分别去乘b(x)
最后再全部异或起来

字

AES除了字节，还有字也是数据处理单位。一个字=4个字节
同时字是在有限域GF(2^8)上，注意一下有8次方，是更大的一个域
所以字的表示方法是，系数取自GF(2^8)上的次数低于4次的多项式
例：字 57 83 4A D1 = 57x3+83x2+4Ax+D1
字也有加法和乘法，加法也是对应系数的模2加
乘法有点差异：b(x)=a(x)c(x) mod x4+1，例子：
a(x)=a3x3 + a2x2 + a1x + a0
b(x)=b3x3 + b2x2 + b1x + b0
c(x)=c3x3 + c2x2 + c1x + c0
所以，c(x)=a(x)b(x) mod x4+1，则以下公式可以直接套用，并且显然是矩阵乘法
c0 = a0 b0 + a3 b1 + a2 b2 + a1 b3
c1 = a1 b0 + a0 b1 + a3 b2 + a2 b3
c2 = a2 b0 + a1 b1 + a0 b2 + a3 b3
c3 = a3 b0 + a2 b1 + a1 b2 + a0 b3

算法描述

输入

明文M，密钥Key

KeyExpansion密钥扩展

像DES准备密钥一样，AES也需要扩展。
将输入的密钥Key扩展为11组128位密钥组，第0组是本身
后面10组中，第n组第i列为第n-1组第i列与第n组第i-1列之和
对于每一组 第0列 即 i=0，有特殊的处理
将前一列即第n-1组第3列（i=3）的4个字节循环左移1个字节，并对每个字节进行字节替代变换SubBytes将第一行（即第一个字节）与轮常量rc[n]相加，最后再与前一组该列相加

State准备

State=M⊕轮密钥（即先执行一次AddRoundKey轮密钥加）

N轮迭代

前r-1轮：这里比DES复杂许多，后面会单独一个个说明

  • SubBytes S盒替换
  • ShiftRows 行移位
  • MixColumns 列混淆
  • AddRoundKey 轮密钥加
    最后第r轮，不做列混淆，做其他三个即可。

输出

此时的State就是密文C

SubBytes S盒替换

由两个可逆变换复合而成

  • 求输入字节A的乘法逆元T
  • 作仿射变换，Y = M · T ⊕ B

在实际编码当中依照如上数学公式，生成S盒，这样每次输入的作为行号和列号，直接就能获取结果
Github上的代码，没有生成部分，直接设置了已经生成的S盒和逆S盒

ShiftRows 行移位

将目标排成矩阵，然后每一行以字节为单位向左位移一定的量即可
第0-3行分别循环左移0-3字节

MixColumns 列混淆

对一个状态逐列进行变换，它将一个状态的每一列视为有限域GF(2^8)上的一个多项式。
在变换中，每一列所表示的多项式将乘以一个固定的多项式C(x)=3 x3+ 1 x2+ 1 x + 2
对应4字节向量为(03 01 01 02)，模多项式为 (x4 + 1)

AddRoundKey 轮密钥加

把对应轮的密钥和目标进行模2加运算，即异或。
###加解密的区别
和DES不同，AES像是DES和倒推的组合
本质上AES解密是整个过程的反向，只是轮密钥加这个动作是再执行一次
密钥使用顺序也是倒着的
如果根据步骤规律重新定义为每一轮的话，那么先执行一次AddRoundKey
前r-1轮：

  • InverseShiftRows 逆向行移位
  • InverseSubBytes 逆向S盒替换
  • AddRoundKey 轮密钥加
  • InverseMixColumns 逆向列混淆

最后第r轮，不做逆向列混淆，做其他三个即可。
实际实现代码当中，除了逆向行移位以外，其他两个Inverse是依靠传矩阵参数来做的。
